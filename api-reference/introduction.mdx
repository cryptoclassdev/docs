## API Overview

The Validator.com API provides programmatic access to validator performance metrics, staking information, and monitoring data. Use our API to integrate validator data into your applications, build custom dashboards, or automate monitoring tasks.

### Base URL

```
https://api.validator.com/v1
```

### Authentication

Most endpoints are public and don't require authentication. For rate-limited endpoints or advanced features, API keys are available upon request.

<Info>
  **API Key**: Contact support@validator.com to request an API key for higher
  rate limits and advanced features.
</Info>

## Quick Start

### Basic Request

```bash
# Get validator status
curl https://api.validator.com/v1/status

# Get staking information for a wallet
curl https://api.validator.com/v1/stake/SOL_WALLET_ADDRESS

# Get performance metrics
curl https://api.validator.com/v1/metrics
```

### Response Format

All API responses are in JSON format:

```json
{
  "status": "success",
  "data": {
    // Response data here
  },
  "timestamp": "2024-01-15T10:30:00Z"
}
```

## Rate Limits

### Free Tier

- **Rate Limit**: 100 requests per minute
- **Burst Limit**: 200 requests per minute
- **Authentication**: Not required
- **Endpoints**: All public endpoints

### Pro Tier

- **Rate Limit**: 1000 requests per minute
- **Burst Limit**: 2000 requests per minute
- **Authentication**: API key required
- **Endpoints**: All endpoints including advanced features

## Error Handling

### HTTP Status Codes

- **200**: Success
- **400**: Bad Request - Invalid parameters
- **401**: Unauthorized - Invalid API key
- **403**: Forbidden - Rate limit exceeded
- **404**: Not Found - Resource not found
- **500**: Internal Server Error - Server error

### Error Response Format

```json
{
  "status": "error",
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "Invalid wallet address format",
    "details": "Wallet address must be a valid Solana address"
  },
  "timestamp": "2024-01-15T10:30:00Z"
}
```

### Webhook Setup

```bash
# Register webhook endpoint
curl -X POST https://api.validator.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.com/webhook",
    "events": ["validator.status", "rewards.distribution"],
    "secret": "your-webhook-secret"
  }'
```

## Examples

### Common Use Cases

<AccordionGroup>
  <Accordion icon="chart-bar" title="Dashboard Integration">
    ```javascript
    // Get validator metrics for dashboard
    const metrics = await api.getMetrics({
      timeframe: '7d',
      granularity: 'hourly'
    });

  </Accordion>
  <Accordion icon="bell" title="Monitoring Alerts">
    ```python

    status = api.get_validator_status()
    
    if status['uptime'] < 99.0:
        send_alert("Validator uptime below 99%")
    
    if status['commission'] != 5.0:
        send_alert("Commission rate changed")
    ```
  </Accordion>
  <Accordion icon="coins" title="Reward Tracking">
    ```go
    // Track staking rewards
    rewards, err := client.GetRewardsHistory(walletAddress, "30d")
    if err != nil {
        log.Fatal(err)
    }
    
    totalRewards := 0.0
    for _, reward := range rewards {
        totalRewards += reward.Amount
    }
    
    fmt.Printf("Total rewards: %.6f SOL\n", totalRewards)
    ```
